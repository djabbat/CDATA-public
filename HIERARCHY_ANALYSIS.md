## Уровень 3 — Молекулярный

### Текущее состояние в CDATA
Все молекулы — **агрегированные скаляры** [0..1]:
- `ros_level` = суммарный ROS (не [O₂⁻], не [H₂O₂], не [OH·] отдельно)
- `protein_aggregates` = агрегированные аgg (не Ub, не p62, не HSP70)
- PTM на центриолях = acetylation + oxidation + methylation + phosphorylation (4 скаляра) → ok!

### Что нужно для EIC-заявки (CAII)
Для валидации нужны 4 **отдельных молекулярных счётчика** дистальных придатков:
```
CEP164_damage: f32    // доминирует в dock для IFT
CEP89_damage:  f32    // стабилизация дистального придатка  
Ninein_damage: f32    // якорь MT к центриоли
CEP170_damage: f32    // удлинение придатка
```
Сейчас все 4 свёрнуты в один `ciliary_function`. Это нужно раскрыть.

### Молекулярные каскады — приоритет для добавления

```
ROS-каскад (детализация):
  O₂⁻ (супероксид)  → через SOD → H₂O₂
  H₂O₂              → через каталазу/GPx → H₂O + O₂
                    → через Фентон (Fe²⁺) → OH· (гидроксил-радикал — самый разрушительный)
  OH·               → окисляет CEP164 > CEP89 > Ninein > CEP170 (разная чувствительность)

Протеасомный каскад:
  poly-Ub цепь → 26S протеасома → деградация
  Если перегружен → аggresomes → аутофагия (p62/SQSTM1)
  Если аутофагия нарушена → накопление → SASP
```

## Уровень 4 — Атомный (Иерархическая термодинамика)

### Концептуальная рамка

Каждый уровень иерархии — **диссипативная структура** (Пригожин):
- Поддерживается потоком энергии (ATP → тепло)
- Имеет собственную скорость производства энтропии σᵢ
- Деградирует при нарушении энергетического потока

### Иерархическая термодинамика для CDATA

```
σ_total = σ_quark + σ_atomic + σ_molecular + σ_organelle + σ_niche

Для биологических систем (упрощение):
σ_atomic ≈ σ_molecular >> σ_organelle >> σ_niche

Тепловое повреждение = k_B × T × ln(Ω_damaged / Ω_intact)
```

**Ключевая идея для CDATA:**  
PTM-накопление = необратимое увеличение энтропии белка центриоля.

```
ΔS_PTM = k_B × ln(число конформационных состояний с PTM /
                   число без PTM)
```

Для CEP164 (~1460 а.к.):
- Ацетилирование Lys: ~50 сайтов → 2⁵⁰ состояний → огромный энтропийный вклад
- Окисление Met/Cys: ~30 сайтов → модифицирует укладку

**Практическое применение в CDATA:**  
Ввести `thermodynamic_damage_rate` как функцию температуры тела (37°C = 310K):
```
damage_rate ∝ exp(-E_activation / k_B × T)
```
При воспалении T→38-39°C → damage_rate растёт экспоненциально.  
Это объяснит inflammaging-ускорение через термодинамику.

### Атомный уровень: конкретные связи

| Химическая связь | Энергия (кДж/моль) | Разрушается OH· |
|---|---|---|
| C–H | 414 | Да (легко) |
| C–C | 347 | Умеренно |
| S–H (Cys) | 339 | Да (тиол→сульфоксид) |
| C=O (карбонил) | 799 | Нет (конечный продукт) |
| N–H | 391 | Да |

**Для CDATA:** карбонилирование белков центриолей (C=O) — **необратимый** маркер потери функции придатков. Уже есть в `oxidation_rate` — но без термодинамического обоснования.

## Уровень 5 — Кварки

### Релевантность для CDATA

Прямая — **ноль** в биологическом контексте.  
Косвенная — через **Ze Vector Theory**:

```
Ze-интерпретация старения:
  Кванты событий T (tension) и S (stretch)
  Здоровая клетка: v ≈ v* ≈ 0.456 (критическая точка)
  Стареющая клетка: v отклоняется от v*
  
  CDATA-Ze связь:
  CAII (appendage integrity) ↔ v (Ze-скорость)
  Полная потеря придатков (CAII=0) → v→0 или v→1 (коллапс)
```

Квантовый уровень → Ze-теория → CDATA: единый параметр v* как термодинамическая точка здоровья.

## Архитектурные выводы для CDATA v2.0

### Что реализовать в порядке приоритета

**Приоритет 1 (для EIC-заявки):**
```
AppendageProteinState {
    cep164_integrity: f32,  // 0=полностью разрушен, 1=интактен
    cep89_integrity: f32,
    ninein_integrity: f32,
    cep170_integrity: f32,
}
// Заменить один ciliary_function на 4 отдельных маркера
// Соответствует прямым измерениям CAII в эксперименте
```

**Приоритет 2 (молекулярный ROS-каскад):**
```
ROSState {
    superoxide: f32,   // O₂⁻
    hydrogen_peroxide: f32,  // H₂O₂
    hydroxyl_radical: f32,   // OH· — главный повреждающий
    iron_pool: f32,    // Fe²⁺/Fe³⁺ — катализатор Фентона
}
```

**Приоритет 3 (термодинамика):**
```
ThermodynamicState {
    local_temperature: f32,  // 37.0 + воспаление
    entropy_production: f32, // σ (дж/моль/с)
    gibbs_free_energy: f32,  // G = H - TS
    damage_rate_multiplier: f32, // exp(-Ea/kT)
}
```

**Приоритет 4 (Гольджи → гликозилирование центриолей):**  
CEP164 имеет N-гликозилирование — нарушение Гольджи → неправильная укладка → ускоренный распад придатков.

**Приоритет 5 (Ze-интеграция):**  
ZeHealthState { v: f32 } — параметр отклонения от v*=0.456, вычисляемый из CAII.

## Временны́е масштабы (важно для шага симуляции)

| Уровень | Характерное время | Шаг симуляции |
|---|---|---|
| Кварк | 10⁻²³ с | — неприменимо |
| Атом (связь) | 10⁻¹⁵ – 10⁻¹² с | — неприменимо |
| Молекула (PTM) | 10⁻³ – 10³ с | 1 час (усреднение) |
| Органелла | 10² – 10⁵ с | 1 день (текущий) |
| Ниша | 10⁶ – 10⁸ с | 1 год (текущий) |

**Вывод:** молекулярный уровень нужно усреднять (mean-field approximation) — моделировать как концентрации, а не отдельные молекулы. Это стандартный подход (ODE/stochastic DE).
